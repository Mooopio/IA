🌐 The Big Picture: How the Web is Structured

Think of the internet like an iceberg. What you see above water (Google, YouTube, news sites) is only a tiny fraction of what exists. The real bulk is hidden below the surface — some of it harmless, some dangerous.

Iceberg Analogy: The surface web is the tip visible above water. The deep web is the massive submerged body. The dark web is the deepest, darkest trench — intentionally hidden, requiring special tools to reach.

The internet is not just one place — it has layers, each with different accessibility, content, and risk levels. Dr. Fahmy organizes these into 6 levels: S, 1, 2, 3, 4, 4B, and 5.

Level S — ClearNet

ClearNet is the fully public, fully indexed layer of the internet. These are websites anyone can reach by typing a URL or searching Google.

  • Social Networks: Facebook, Twitter, Instagram
  • Search Engines: Google, Bing, Yahoo
  • Wikis: Wikipedia
  • Email services: Gmail, Outlook
  • Common internet content: news, blogs, shopping
No special tools needed. Just a normal browser.
Level 1 — Surface Web

The Surface Web is the primary entry point for most users. It contains all websites whose data is indexed by search engines like Google.

  • Amazon, eBay (e-commerce)
  • Blogs and news sites
  • University public databases
  • Hosting services and forums
  • Closed social networks (requiring login but still indexed)
Key Fact: Despite being the most visible layer, the Surface Web contains very little of the internet's total data. Most data sits deeper.

Access method: Standard browsers (Chrome, Firefox, Safari). No special tools required.

Level 2 — Bergie Web

A transitional layer between the publicly accessible web and the hidden web. Still reachable via normal browsers, but with restricted or semi-hidden content.

  • Web Archive (archive.org — old snapshots of sites)
  • Torrents & P2P file sharing
  • Streaming services (some require subscriptions)
  • Restricted Access Content
  • Robots.txt — files that tell crawlers what NOT to index
  • Anti-Virus Databases
Robots.txt analogy: Imagine a "No Photography" sign at a museum. The museum is open to the public, but you're asked not to photograph certain exhibits. Robots.txt tells search engines: "don't index this page," but the page itself isn't secret.
Level 3 — Deep Web

The Deep Web is the largest part of the internet by far. It is NOT inherently dangerous or illegal — it's simply not indexed by standard search engines.

📊 Key Statistics (Exam Favorite!)

400–500 times more information than the Surface Web.
15% larger visit capacity than the Surface Web.

Why Is It Not Indexed?

  • Pages are dynamically generated from databases — they don't have fixed URLs that crawlers can follow
  • Access control required: login, authentication, subscription
  • Data is behind forms (search queries generate results that aren't indexable)
  • Content is blocked by robots.txt

Common Deep Web Content

  • Medical records (your doctor's patient portal)
  • Financial records (your bank account)
  • Government databases
  • Academic journals (behind paywalls)
  • Corporate intranets
Library Analogy: The Surface Web is like the open shelves. The Deep Web is like the archives — not secret, but you need to ask the librarian (authenticate) or know the exact location (URL) to access it.

Information Extraction (IE) Categories — EXAM CRITICAL!

Since deep web data is in database records, extracting it is called Information Extraction (IE). There are 3 types:

IE TypeScopeWhat it does
Record-level IEIndividual recordsDiscovers record boundaries and divides data into separate attributes (e.g., name, address, phone)
Page-level IEOne web pageExtracts ALL data embedded in a single web page
Site-level IEEntire websitePopulates a database from all pages of a web site

Deep Web & Meta Search Engines

Deep web pages represent data as data records. These records are useful in meta search engines, but must first be extracted. This extraction = Information Extraction.

Even Google only accesses a very small part of the Deep Web. Most users cannot find deep web information just by typing keywords.

Access Method

Use TOR (The Onion Router) — a special browser. TOR can also access Deep Web content not available through surface browsers, including Hidden WIKI and Bitcoin services.

Deep web URLs often end in .onion — e.g., http://SiteName.onion/

Deep Web ≠ Dark Web. Deep Web = large, mostly benign. Dark Web = small, largely illegal subset of Deep Web.
Level 4 — Dark Web

The Dark Web is a small subset of the Deep Web that is intentionally hidden and requires special software to access.

Key Characteristics

  • Part of the web NOT indexed by search engines
  • Requires TOR or I2P software
  • This software connects you to an overlay network
  • Your IP address is masked behind layers of other IPs — untraceable

How TOR Works (Conceptually)

Onion Analogy: TOR wraps your data in multiple layers of encryption (like an onion). Each relay server peels off one layer and forwards it, without knowing the full path. Nobody sees the whole picture.

Dark Web Content (Exam Loves This List!)

  • ⚔️ Violence & Weapons
  • 💊 Illicit Drugs (e.g., SilkRoad marketplace)
  • 💻 Hacking groups and guides
  • 📁 Banned Media
  • 🔗 Illicit Links
  • 🔞 Illicit Pornography
  • ☠️ Extremism
  • 💰 Illicit Finance & Crypto Services
  • 🧅 Hidden Wiki & most .onion sites
  • 🛒 Drug Dealers (SilkRoad)
Accessing the Dark Web is illegal or heavily regulated in many countries. Content here is extremely dangerous and illegal.
Level 4B — Private Web

A classified, restricted-access layer beyond the regular Dark Web. Contains government and research systems.

  • FPI Mid-Classified data
  • Supercomputing networks
  • Closed Systems
  • Geometric algorithm research
Level 5 — Marianas Web

Named after the Mariana Trench — the deepest, most inaccessible part of the ocean. Called the "final boss of the web."

Considered the most dangerous and harmful layer. The material strongly advises to stay away.

Little is publicly known about its contents, but it is portrayed as the ultimate, unreachable depth of the internet.

Deep Web

  • Contains 400–500× more data than surface web

  • Pages are dynamically generated from databases
  • Not indexed because crawlers can't follow dynamic links

  • Small part of the Deep Web
  • Requires TOR or I2P
  • Creates an overlay network
  • Masks IP through layers of other IPs (untraceable)
  • Content: drugs, weapons, hacking, illicit media, extremism

🚨 Common Exam Traps
  • ❌ "Deep Web = Dark Web" — WRONG! Dark Web is a small subset of Deep Web
  • ❌ "Deep Web is illegal" — WRONG! It contains your bank account and medical records
  • ❌ "TOR is only for Dark Web" — TOR can also access parts of the Deep Web
  • ❌ "Surface Web = all of the internet" — Surface Web is actually a tiny portion
  • ✅ I2P is an alternative to TOR, not the same software
  • ✅ Robots.txt appears at Level 2 (Bergie Web), not Deep Web

🔑 Key Definitions

Surface Web: Indexed websites, normal browsers
Deep Web: Not indexed; dynamically generated; legal content
Dark Web: Subset of deep web; TOR/I2P; illegal content
TOR: The Onion Router — masks IP via layered encryption
I2P: Alternative to TOR for anonymous browsing
Overlay Network: Network built on top of another network
Information Extraction: Process of pulling structured data from deep web pages

📊 Critical Numbers

Deep Web = 400–500× more data than Surface Web
Deep Web = 15% larger visit capacity
Dark Web = small part of Deep Web
Surface Web = tiny fraction of total internet

🗂️ Layer Quick Reference

S: ClearNet — Google, FB, email
1: Surface — Amazon, blogs, uni DB
2: Bergie — Archives, torrents, robots.txt
3: Deep — Medical/financial records, not indexed
4: Dark — TOR, .onion, SilkRoad, Hidden Wiki
4B: Private — FPI, supercomputing, closed
5: Marianas — "Final boss", stay away

🔬 IE Types

Record-level: Finds record boundaries → attributes
Page-level: All data from ONE web page
Site-level: Populates DB from entire site

⚠️ Key Distinctions

Deep Web ≠ Dark Web
Deep Web is NOT illegal
Dark Web IS (mostly) illegal
Surface ≠ ClearNet (ClearNet is Level S, Surface is Level 1)
.onion = Dark/Deep Web URL format
