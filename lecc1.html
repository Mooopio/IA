The World Wide Web

INTERNET vs WEB — مش نفس الحاجة!

DEFINITION

The World Wide Web is a system of hypertext documents that are viewable by a web browser. It runs as a service/application on top of the Internet protocols.

🗣️ بالمصري الإنترنت ده زي الطريق السريع — شبكة ضخمة من الكابلات والاتصالات.

الـ Web مش هو الإنترنت! الـ Web ده تطبيق بيشتغل فوق الإنترنت. زي ما الأتوبيس بيشتغل فوق الطريق — الطريق هو الإنترنت، الأتوبيس هو الـ Web.

Hypertext يعني إيه؟ يعني نص فيه لينكات توصلك لوثائق تانية — زي لما بتضغط على لينك في صفحة ويب توصلك لصفحة تانية.
⚡ The 3 Fundamental Mechanisms of the Web
  • HTTP (Hypertext Transfer Protocol) — بروتوكول التواصل بين المتصفح والسيرفر
  • HTML (Hypertext Markup Language) — لغة وصف الوثائق
  • URI (Uniform Resource Identifiers) — عنوان أي مورد على الإنترنت
🗣️ تصور الموضوع عايز تبعت رسالة لصاحبك؟
✉️ الـ URI = عنوان صاحبك (العنوان الإلكتروني)
📬 الـ HTTP = نظام البريد (الطريقة اللي الرسالة بتتبعت بيها)
📝 الـ HTML = محتوى الرسالة نفسها (شكل الكلام والتنسيق)
🚀

Web 2.0

The Social & Interactive Web

DEFINITION

Web 2.0 is a buzzword for modern forms of applications and services accessible by browser such as wikis or weblogs. They have a high social component, appealing user interfaces, and enhanced interaction models.

🗣️ بالمصري الويب 1.0 كان بتدخل وتقرأ بس — زي ما تقرأ جرنال.
الـ Web 2.0 بقى تقدر تتفاعل — تكتب تعليق، تشارك، تعمل محتوى بنفسك. فيسبوك، يوتيوب، ويكيبيديا — دي كلها Web 2.0.
الميزة التقنية المستخدمة الشرح
Increasing client-side logic JavaScript, Ajax, WebSockets المنطق بدأ ينزل عند المستخدم مش على السيرفر بس
New document formats (X)HTML, RSS, Atom, RDF, HTML5 صيغ جديدة لتمثيل البيانات
Protocol improvements Ajax, Comet/HTTP Streaming, WebSockets طرق أسرع وأحدث للتواصل
Social communities Social Software شبكات اجتماعية ومجتمعات
New business models — نماذج تجارية جديدة ظهرت مع الويب
⚠️ مهم جداً!

الـ Lecture بيقول Web 2.0 هي technological evolution (تطور تكنولوجي) — يعني الأدوات اتغيرت. لكن Web 3.0 هي conceptual evolution (تطور في الفكرة).

🔮

Web 3.0 / Semantic Web

The Web that Machines Can Understand

DEFINITION

Web 3.0 is a vision of semantically interconnected data. Focus is not on documents but on data — thus becoming Linked Data. Key formats: RDF, RDFS, OWL, Microformats.

🗣️ بالمصري دلوقتي لو بحثت عن "مباريات النهارده"، الكمبيوتر بيفهم كلمة "مباريات" ككلمة بس مش كمفهوم رياضي.

الـ Semantic Web بيقول: خليني أفهم معنى البيانات، مش بس الكلام. الكمبيوتر يفهم إن "القاهرة" هي عاصمة مصر وفيها تعداد سكاني كذا، وهي مرتبطة بـ"مصر" كدولة — مش بس كلمة.

ده اللي Tim Berners-Lee (مخترع الويب) حلم بيه: آلات تتكلم مع بعض وتفهم البيانات.
💡 الفرق الجوهري

Web 1.0: اقرأ بس — Read only
Web 2.0: اقرأ واكتب — Read + Write (Social)
Web 3.0: اقرأ واكتب والآلات تفهم — Read + Write + Machine Understanding

XML — ما هو وليه مهم

Extensible Markup Language

DEFINITION

XML is a general purpose meta-language for representation, exchange and processing of data. XML documents are organised in a simple tree structure with elements as nodes.

🗣️ بالمصري XML مش لغة برمجة — هي لغة لوصف البيانات. متخيلهاش زي Python أو Java. خيلها زي اللوحة اللي بتكتب عليها بياناتك بشكل منظم يأي برنامج في أي لغة يفهمه.

كلمة meta-language يعني: هي لغة بتعرّف لغات تانية! خلتنا نبني HTML, XHTML, RSS, Atom, RDF كلهم فوقيها.

🌳

XML Tree Structure & Terminology

Root, Parent, Child, Sibling, Attribute, Content Types
🗣️ بالمصري — هيكل شجرة XML XML بتتنظم زي شجرة عيلة! فيه:
🌱 Root Element: أصل الشجرة — element جامع واحد بس يحتوي على الكل
👪 Parent/Child: كل element ممكن يحتوي على elements جوه (أولاده)
👫 Siblings: الـ elements اللي في نفس المستوى وتحت نفس الـ parent
🏷️ Attribute: معلومة إضافية داخل الـ tag زي id="42" 
<?xml version="1.0" encoding="UTF-8"?>

<bookstock>               <!-- ROOT ELEMENT -->
  <book id="42">           <!-- CHILD of bookstock, PARENT of title/author/isbn -->
    <title>TCP/IP Illustrated</title>    <!-- SIBLING of author -->
    <author>W.R. Stevens</author>     <!-- Simple Content (text) -->
    <isbn>0201633469</isbn>
  </book>
  <book id="43">
    <title>Mobile Web Services</title>
  </book>
</bookstock>
<!-- End of document -->
🗣️ أنواع الـ Content 1️⃣ Empty Content: مفيش حاجة جوه <br/>
2️⃣ Simple Content: نص عادي <title>TCP/IP</title>
3️⃣ Element Content: بيحتوي elements تانية <book><title/></book>
4️⃣ Mixed Content: نص وعناصر مع بعض

XML Syntax Rules — Well-Formed

قواعد لازم تتحفظها غيباً!
🗣️ بالمصري — قواعد XML الإلزامية لو مش بتحترم القواعد دي، الـ XML مش هيتقرأش! لازم الوثيقة تكون Well-Formed.
القاعدة صح ✅ غلط ❌ الشرح
كل tag له close tag <X>content</X> <X>content مش زي HTML اللي بيسمح بحاجات مفتوحة
Empty element ليه شكل مخصوص <X/> <X></X> (مقبول لكن) الشكل المختصر للعنصر الفاضي
Root element واحد بس <root>...</root> <A/><B/> (في نفس المستوى) لازم يكون في عنصر واحد جامع
Attributes بين quotes id="42" id=42 Quotes إلزامية، سواء single أو double
Tags مش بتتشابك <X><Y></Y></X> <X><Y></X></Y> لازم تقفل الجواني الأول
Case Sensitive <Book>...</Book> <Book>...</book> كبير وصغير مختلفين تماماً!
⚠️ الفرق بين Well-Formed و Valid

Well-Formed: بيحترم قواعد XML الأساسية
Valid: Well-Formed + بيتوافق مع Schema أو DTD محددة

كل Valid document هو well-formed، بس مش كل well-formed document هو valid!

XML Namespaces

حل مشكلة تعارض الأسماء
🗣️ المشكلة تخيل عندك ملف XML بيجمع بيانات عن كتاب وعن بيانات الملف نفسه. الاتنين فيهم عنصر اسمه <title>. لما الكمبيوتر يدور على "title"، يجيب إيه؟ 😱 
<!-- المشكلة -->
<datafile>
  <title>Book list</title>        <!-- title الملف -->
  <book>
    <title>TCP/IP Illustrated</title> <!-- title الكتاب -->
  </book>
</datafile>

<!-- الحل: Namespaces -->
<datafile xmlns:a="http://www.example.com/doc/"
          xmlns:b="http://www.example.com/myNs/">
  <a:title>Book list</a:title>        <!-- namespace "a" -->
  <b:book>
    <b:title>TCP/IP Illustrated</b:title> <!-- namespace "b" -->
  </b:book>
</datafile>
🗣️ بالمصري الـ Namespace زي اللقب. عندك في الشغل محمد أحمد ومحمد خالد — الاتنين اسمهم محمد. لو قلت "محمد" مش هيعرفوا مين. لكن لو قلت "محمد أحمد" — اتحدد!

الـ xmlns:a = بتعرّف prefix اسمه "a" وبتربطه بـ URL (الـ URL ممكن ميوصلش أي حاجة، بس بيعمل identifier فريد).
⚠️ نقطة مهمة جداً!

الـ namespace URL (زي http://www.example.com/doc/) مش لازم يوصل لحاجة حقيقية! هو بس identifier — زي الرقم القومي، بيعرّف بس.

XML Validity: DTD vs XML Schema

إزاي بنتحقق إن الـ XML صح في الشكل والبيانات
🗣️ بالمصري Well-formed = البنية صح.
Valid = البنية صح وكمان البيانات بتطابق قواعد محددة اتحددت في DTD أو XML Schema.

زي استمارة حكومية: محتاج تملأها صح (well-formed) وكمان البيانات اللي بتكتبها لازم تكون من نوع معين (valid).
الخاصية DTD XML Schema
لغته لغة خاصة مختلفة XML نفسه! (dialect)
أنواع البيانات محدودة جداً (PCDATA بس) غنية: string, integer, date, …
Namespaces دعم محدود دعم كامل
الاستخدام قديم، أبسط الأحدث والأقوى 
<!-- DTD Example -->
<!DOCTYPE note [
  <!ELEMENT book (title,author,isbn)>
  <!ELEMENT title (#PCDATA)>   <!-- PCDATA = نص عادي فقط -->
  <!ELEMENT author (#PCDATA)>
  <!ELEMENT isbn (#PCDATA)>
]>

<!-- XML Schema Example -->
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema">
  <xs:element name="book">
    <xs:complexType>
      <xs:sequence>
        <xs:element name="title"  type="xs:string"/>
        <xs:element name="author" type="xs:string"/>
        <xs:element name="isbn"   type="xs:string"/>
      </xs:sequence>
    </xs:complexType>
  </xs:element>
</xs:schema>
🗣️ الـ XML Schema أقوى ليه؟ في DTD، كل حاجة بتبقى PCDATA (نص). في XML Schema تقدر تقول:
— هذا الحقل integer (رقم صحيح)
— هذا الحقل date بتاريخ معين
— هذا الحقل string طوله بين 5 و 20 حرف
أقوى بكتير في التحقق من صحة البيانات!

XHTML

XML Reformulation of HTML

DEFINITION

XHTML is an XML reformulation of HTML. Stricter syntax + standard XML tools work on it + extensible with other XML languages like MathML.

🗣️ بالمصري HTML كانت متساهلة — تقدر تكتب tag مش مقفول، attribute من غير quotes، وكمان المتصفح هيشتغل.

XHTML = HTML بس بقواعد XML الصارمة. يعني:
✅ كل tag لازم مقفول (<br/> مش <br>)
✅ كل attribute بين quotes
✅ تقدر تستخدم أدوات XML العادية عليها (parsers, validators, …)
✅ تقدر تمدها بـ MathML و SVG وغيرها 
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN"
  "http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">

<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="en">
<head>
  <title>XHTML Example</title>
</head>
<body>
  <h3>Available books</h3>
  <br/>  <!-- Empty elements MUST be closed! -->
  <table>
    <tr><td>Title</td><td>Author</td></tr>
  </table>
</body>
</html>

DOM — Document Object Model

Tree-Based XML Processing

DEFINITION

DOM is a W3C-standardised API for accessing and manipulating XML documents. It loads the entire document as a tree in memory, allowing traversal and direct element access by ID.

🗣️ بالمصري الـ DOM بيقرأ الملف كله وبيعمله شجرة في الـ RAM. زي ما تاخد كتاب وتحفظ تقريباً كل حاجة فيه في دماغك — تقدر بعدين تتنقل وتدور في أي حتة بسهولة.

ميزة: سهل تتنقل فيه وتعدّل في أي مكان
عيب: لو الملف كبير جداً، الذاكرة هتتملى! 
// DOM JavaScript API
var elem = document.getElementById("42");
var child = elem.getElementsByTagName("author")[0];
var secondChild = child.childNode;
var value = secondChild.nodeValue;
⚡

SAX — Simple API for XML

Event-Based XML Processing

DEFINITION

SAX is an event-based mechanism for XML processing. A program registers event handlers at a SAX parser. The parser reads the document sequentially and fires events for each element, attribute, and character.

🗣️ بالمصري — SAX vs DOM الـ DOM = بتقرأ الكتاب كله وبعدين بتدور فيه.
الـ SAX = زي ما تقرأ كل سطر وتعمل حاجة فوراً وبعدين تمشي — ما بتحفظش الكتاب!

ميزة SAX: أسرع بكتير للملفات الكبيرة ومش بياكل RAM
عيب SAX: ما تقدرش ترجع لجزء سبق، وصعب تعدّل في البيانات
🗣️ إزاي SAX بيشتغل؟ 1️⃣ برنامجك بيسجّل Event Handlers عند SAX Parser
2️⃣ الـ Parser بيقرأ الملف سطر بسطر
3️⃣ كل ما يلاقي element أو attribute أو نص — بيطلق Event
4️⃣ الـ Event Handler بتاعك بيشتغل وبيعمل اللي انت عايزه 
// SAX Handler Events
public void startDocument() { // اتفتح الملف
    System.out.println("Start document");
}
public void startElement(...) { // لقينا <tag>
    ...
}
public void characters(...) { // محتوى نصي
    ...
}
public void endElement(...) { // لقينا </tag>
    ...
}
public void endDocument() { // انتهى الملف
    System.out.println("End document");
}
المقارنة DOM SAX
طريقة العمل Tree in memory Event-based, sequential
استهلاك الذاكرة عالي (الملف كله) منخفض جداً
السرعة أبطأ للملفات الكبيرة أسرع
التعديل سهل جداً صعب/مستحيل
العودة لجزء سابق ممكن مستحيل

CSS — Cascading Style Sheets

Client-side layout language for markup elements

DEFINITION

CSS is a client-side language for defining the layout of markup language elements. Can be embedded in HTML or linked as external file — enabling one CSS file to control the layout of all associated pages.

🗣️ بالمصري الـ HTML بيقول "فيه heading وباراجراف وجدول".
الـ CSS بيقول "الـ heading لونه أحمر وكبير، والباراجراف فونته Garamond".

أهم ميزة: لو عندك 100 صفحة ويب وعايز تغير لون الـ background — بتعدل ملف CSS واحد بس وكل الصفحات بتتغير!
🗣️ أنواع الـ Selectors 🔵 Tag Selector: p { ... } — بيأثر على كل <p> في الصفحة
🟡 Class Selector: .top { ... } — بيأثر على أي element فيه class="top"
🔴 ID Selector: #fat { ... } — بيأثر على element معين بـ id="fat" فقط 
/* main.css */
input {
    font-size: 110%;
    color: red;
    background: #ffff00;
}

.top {                   /* Class selector */
    padding-top: 20px;
}

p {
    font-size: 16pt;
    font-family: "Garamond", serif;
    font-weight: bold;
}

#fat {                  /* ID selector */
    font-weight: bold;
}
⚙️

JavaScript

Client-side scripting + Sandbox principle

DEFINITION

JavaScript is an object-based script language (standardised as ECMAScript), often used for client-side execution. It may be embedded in HTML or linked from it. The JavaScript engine executes it and the result can influence the document's content.

🗣️ بالمصري الـ HTML و CSS بيعملوا الشكل. JavaScript بيعمل الحركة والتفاعل.
لما بتضغط زرار ويحصل حاجة من غير ما الصفحة تتحمل من جديد — ده JavaScript.

Client-side يعني بيشتغل في المتصفح عند المستخدم، مش على السيرفر.
⚠️ Sandbox Principle — مهم جداً للامتحان!

الـ JavaScript بيشتغل في sandbox — بيعني إنه ما يقدرش يوصل لـ:
❌ File System (ملفات الكمبيوتر)
❌ Network resources خارج المتصفح (تحت قيود)
✅ Objects داخل المتصفح بس (DOM, window, …)

ده لأمان المستخدم — لو مكانش في sandbox، أي موقع يزوره يقدر يقرأ ملفاتك!

🗣️ JavaScript + DOM الكومبو الشهير: JavaScript + DOM.
JavaScript بتستخدم DOM API عشان تعدّل في الـ HTML ديناميكياً — تضيف element، تشيل، تغير لون — من غير ما تعيد تحميل الصفحة.

YAML & Content Types

Alternatives to XML + MIME Types

YAML

YAML Ain't Markup Language — compact, human-readable data serialization format. It is a superset of JSON. Uses indentation for hierarchies instead of tags. Offers lists, associative arrays, and scalars.

🗣️ بالمصري XML فيها overhead كتير — كل بيانة محاطة بتاج افتتاح وتاج إغلاق. YAML أخف وزناً وأسهل قراءة.
الـ Indentation (المسافات) بيعمل نفس دور الـ Tags في XML. 
# YAML Example (بدل XML)
lectures:
  - 'Internet and Web Applications'
  - 'Distributed Systems'
  - 'Mobile Communication and Computing'

# نفس الحاجة في XML:
<lectures>
  <lecture>Internet and Web Applications</lecture>
  <lecture>Distributed Systems</lecture>
</lectures>

Content Types (MIME Types)

Format: Content-Type: type/subtype
Used in HTTP headers to tell the client what kind of content it received, so it can find the right interpreter.

🗣️ بالمصري السيرفر لما يبعتلك ملف، بيقولك نوعه في الـ Header. المتصفح بيقرأ النوع ده ويبعته لبرنامج مناسب.
زي ما لو اشتريت حاجة أونلاين وعليها كتابة "هش — لا ترمي" — الـ content type هو ده الكتابة دي.
Content-Type النوع
application/xmlملف XML عام
text/htmlصفحة HTML
image/jpegصورة JPEG
video/mpegفيديو MPEG
text/plainنص عادي

الحاجات اللي بتلغبط الناس

Common Exam Traps & Hard Concepts
🪤 فخ 1: Internet ≠ Web HARD

الإنترنت هو الشبكة التحتية (TCP/IP, routers, cables, …)
الـ Web هو تطبيق واحد يشتغل فوق الإنترنت (زي Email, FTP, BitTorrent — كلها تطبيقات على الإنترنت).
السؤال في الامتحان: "Does the Web = the Internet?" → لا!

🪤 فخ 2: Well-Formed vs Valid HARD

Well-Formed = XML syntax صح (tags مقفولة، root واحد، …)
Valid = Well-formed + بيتطابق مع DTD أو Schema
كل valid document هو well-formed، بس العكس مش صح!

🪤 فخ 3: Namespace URL مش URL حقيقي MEDIUM

لما بتكتب xmlns:a="http://www.example.com/doc/" — الـ URL ده ما بيروحش لحاجة حقيقية على الإنترنت في الغالب!
هو بس identifier فريد — زي رقم قومي، مهمته التمييز فقط.

🪤 فخ 4: SAX vs DOM — إمتى تستخدم إيه؟ HARD

SAX: ملفات كبيرة، محتاج تقرأ بس (read-only), memory مش كتير
DOM: محتاج تتنقل في الملف، تعدّل، تدور في أي مكان
SAX ما بيبنيش tree في الذاكرة — de كل الفرق

🪤 فخ 5: XML Case Sensitive تماماً EASY

<Book> مختلف عن <book> مختلف عن <BOOK>
<Book>content</book> = ERROR!
أي سؤال فيه خلط بين كبير وصغير — الإجابة غلط.

🪤 فخ 6: Web 2.0 = Technological, Web 3.0 = Conceptual MEDIUM

Web 2.0 = technological evolution (أدوات وتقنيات جديدة)
Web 3.0 = conceptual evolution (فكرة جديدة عن معنى البيانات)
دي جملة من الـ Lecture نفسه — بتيجي في الامتحان!

🪤 فخ 7: JavaScript Sandbox HARD

JavaScript في المتصفح ما بيقدرش يوصل لـ File System أو Network Resources خارج المتصفح بشكل مباشر.
ده مش bug — ده feature أمني اسمه Sandbox Principle.

🪤 فخ 8: DTD مش XML MEDIUM

الـ DTD بيتكتب بـ syntax مختلفة خالص — مش XML!
الـ XML Schema هو اللي بيتكتب بـ XML.
ده فرق مهم لو الامتحان سأل "أي منهم يستخدم XML syntax؟" → XML Schema فقط.

📋 Exam Summary — Internet & Web Applications Lecture 1

🌐 1. Web Fundamentals

  • The Web is NOT the Internet — it's an application running ON TOP of Internet protocols.
  • The Web is built on three pillars: HTTP (protocol), HTML (document format), URI (addressing).
  • Web 2.0 = technological evolution — social features, Ajax, WebSockets, user-generated content.
  • Web 3.0 = conceptual evolution — Semantic Web, Linked Data, machine-understandable data. Key formats: RDF, RDFS, OWL.

📄 2. XML Core Concepts

  • XML is a general-purpose meta-language for data representation, exchange, and processing.
  • XML documents are organized as trees with elements as nodes.
  • Content types: empty, simple (text), element (nested tags), mixed.
  • A Well-Formed document obeys XML syntax rules: one root, closed tags, quoted attributes, no overlapping tags, case-sensitive.
  • A Valid document is well-formed AND conforms to a DTD or XML Schema.

🏷️ 3. XML Namespaces

  • Namespaces solve name conflicts by qualifying elements with unique identifiers (URLs).
  • The namespace URI does NOT need to point to a real resource — it's just a unique identifier.
  • Syntax: xmlns:prefix="URI" then use prefix:elementName.

✅ 4. DTD vs XML Schema

  • DTD: Uses its own syntax (NOT XML), limited type support (only PCDATA), older.
  • XML Schema: Written in XML itself, rich data types (string, integer, date…), supports namespaces fully.
  • XML Schema defines: Simple types (string, integer…) and Complex types (sequences of other types).

🏗️ 5. XHTML

  • XHTML = HTML reformulated as XML — stricter syntax, processable by standard XML tools.
  • All tags must be closed (including empty ones: <br/>), attributes must be quoted, case-sensitive.
  • Can be validated by DTD (e.g., XHTML 1.1 DTD).
  • Extensible: can embed MathML, SVG, etc.

🌲 6. DOM vs SAX

  • DOM: Loads entire XML as tree in memory. Good for traversal and manipulation. High memory usage.
  • SAX: Event-based, sequential parsing. Low memory, fast, but no tree — cannot go back or modify easily.
  • Key DOM methods: getElementById(), getElementsByTagName(), childNode, nodeValue.
  • Key SAX events: startDocument(), startElement(), characters(), endElement(), endDocument().

🎨 7. CSS

  • Client-side language for defining layout of markup elements.
  • Can be external (linked), embedded, or inline.
  • Selectors: Tag (p {}), Class (.top {}), ID (#fat {}).
  • Single external CSS file can control layout of all associated pages.

⚙️ 8. JavaScript

  • Object-based scripting language, standardised as ECMAScript.
  • Runs client-side in the JavaScript engine inside the browser.
  • Sandbox principle: can only access browser objects by default — no direct file system or network access.
  • Often used with DOM API to dynamically change document structure.

📦 9. Content Types (MIME Types)

  • Format: Content-Type: type/subtype (e.g., application/xml, image/jpeg, text/plain).
  • Sent in HTTP headers to tell the client what type of content it received.
  • Client uses it to find the right interpreter/application for the content.

🔁 10. YAML (Alternative to XML)

  • YAML Ain't Markup Language — compact, human-readable data serialization.
  • Superset of JSON. Uses indentation for hierarchy (no tags).
  • Supports: lists, associative arrays, scalars, cross-references.

⚡ Quick-Fire Exam Facts

  • Web 2.0 = technological evolution; Web 3.0 = conceptual evolution.
  • XML namespace URLs do NOT need to point to real resources.
  • Well-Formed ≠ Valid. All Valid XML is Well-Formed, not vice versa.
  • XML is case-sensitive. <Book> ≠ <book>.
  • DTD uses non-XML syntax; XML Schema uses XML syntax.
  • SAX does NOT build a tree in memory.
  • JavaScript sandbox: no direct file system access by default.
  • Tim Berners-Lee invented the original Web and envisioned Web 3.0.
