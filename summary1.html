1 The World Wide Web – 3 Pillars
Protocol

HTTP

HyperText Transfer Protocol — the application protocol used for communication between client and server. Client fetches documents from server using HTTP.

Format

HTML

HyperText Markup Language — the document description format. Hypertext documents have internal cross-references to other documents.

Addressing

URI

Uniform Resource Identifier — the document addressing mechanism. Every resource on the web is identified by a URI.

2 Client–Server Interaction Flow
1️⃣ Client (browser) requests a document identified by a URI via HTTP
2️⃣ Server returns the document
3️⃣ Browser interprets the document — it may contain: XHTML (text/structure), CSS (layout/style), JavaScript (program logic)
Document Layer

XHTML — Displayed Data

Carries actual text content and structure of the page.

Style Layer

CSS — Layout & Style

Defines visual presentation: fonts, colors, spacing.

Logic Layer

JavaScript — Program Logic

Handles interactivity, dynamic changes to DOM.

3 Content Type (MIME Type)
Why it exists: The client needs to find an appropriate interpreter for received content. Every HTTP response includes a Content-Type header.
Format

Content-Type: type/subtype

  • application/xml — general XML content
  • video/mpeg — MPEG encoded video
  • image/jpeg — JPEG image
  • text/plain — plain text
  • text/html — HTML document
📋 How the client handles unknown types: If the client (browser) is NOT responsible for the content type, it looks up an appropriate application in its database and delegates the document to that application.
1 XML — What & Why
XML = Extensible Markup Language
A general-purpose meta-language for representation, exchange and processing of data. It is the basis for many web-related formats.
Built on XML

XML-based Formats

  • XHTML
  • XSL (Extensible Stylesheet Language)
  • XML Schema
  • RSS (Really Simple Syndication)
  • Atom
  • RDF (Resource Description Framework)
Structure

Tree Structure

XML documents are organised in a simple tree structure with the document's elements as nodes.

bookstock ├── book id="42" │ ├── title │ ├── author │ └── isbn └── book id="43" ├── title └── author (×2 siblings)

3 XML Syntax Rules (Well-Formed)
Syntax

Must Follow These Rules

  • Non-empty elements: must have start AND end tag <X>…</X>
  • Empty elements: can self-close <X/>
  • Exactly one root element
  • All attribute values must be quoted (single or double)
  • Nested tags must NOT overlap — <X><Y></X></Y> is invalid
  • Case sensitive — <X></x> is wrong
  • Content must comply with declared charset (default: UTF-8)
XML Declaration

Document Header

<?xml version="1.0" encoding="UTF-8"?> <!-- comment here --> <bookstock> <book id="42"> <title>TCP/IP Illustrated</title> </book> </bookstock>
4 XML Namespaces
⚠️ Problem: Different contributors may use the same element name (<title>) to refer to different things. Mixing them causes name conflicts.
Solution

Namespaces qualify elements with unique identifiers

  • Identifiers are URIs (web addresses — may point to non-existing resources)
  • A prefix is bound to a namespace: xmlns:a="http://…"
  • All elements in that namespace get the prefix: <a:title>
  • Different namespaces → different prefixes → no conflicts
  • Standardised XML dialects define a namespace imported by all documents using that dialect
<datafile xmlns:a="http://example.com/doc/" xmlns:b="http://example.com/myNs/"> <a:title>Book list</a:title> <!-- doc title --> <b:title>TCP/IP Illustrated</b:title> <!-- book title --> </datafile>

6 XHTML
XHTML = Extensible HyperText Markup Language
An XML reformulation of HTML. Language constructs deduced from HTML but with XML's strict syntax.
Why XHTML?

Key Advantages

  • High standardisation → unification of browser engines
  • Stricter syntax than HTML
  • Processable by regular XML tools (style, validation)
  • Extensible by other XML languages (e.g. MathML for math expressions)
  • Can be validated by DTD (e.g. XHTML 1.1)
XHTML Rules (stricter than HTML)

Notable Differences

  • Must have XML declaration
  • Must reference DTD (<!DOCTYPE html …>)
  • Must define default namespace (xmlns="…")
  • Empty elements must self-close: <br/>
  • All tags lowercase, all attributes quoted

2 DOM — Deep Dive
DOM is an API for accessing and manipulating XML documents. It interprets the document as a logical tree structure — you must load the whole tree into memory.
Key Operations

DOM API Methods

// Access element by ID var elem = document .getElementById("42") // Get children by tag name var child = elem .getElementsByTagName("author")[0] // returns a LIST of elements // Navigate to child node var second = child.childNode // Get text value var val = second.nodeValue
Traversal

Tree Navigation

  • Can traverse through the tree (parent → child → sibling)
  • Can directly access elements by id
  • getElementsByTagName() returns a list of matching elements
  • Works in conjunction with JavaScript to dynamically modify pages
3 SAX — Deep Dive
SAX = event-based mechanism for XML processing. The parser reads sequentially; every time it encounters an element/attribute, it fires an event to a registered handler.
SAX 3-Step Process

How SAX Works

  • Step 1: Program registers event handlers at a SAX parser
  • Step 2: Parser reads XML document sequentially from start to end
  • Step 3: Each element/attribute/character → parser fires an event → handler extracts wanted data
SAX Handler Events

Event Types (Java example)

void startDocument() { … } void startElement(…) { … } void characters(…) { … } void endElement(…) { … } void endDocument() { … }

1 JavaScript
JavaScript is an object-based script language standardised as ECMAScript. Mainly used for client-side execution in web browsers.
How It Works

Embedding & Execution

  • Can be embedded directly in (X)HTML or linked from an external file
  • Browser identifies code segments → forwards to JavaScript engine
  • Execution result may: modify the document OR force a reaction (e.g. open new window)
<script language="JavaScript" type="text/javascript"> function hello() { alert("Hello!"); } </script> <body onLoad="hello()">
Security

Sandbox Principle

  • JS runs inside the JavaScript engine (sandbox)
  • By default: can only access objects inside the browser
  • NO direct access to: file system, external network resources
  • Protects the user's machine from malicious scripts
JS + DOM

Dynamic Pages

  • JS is often used in combination with DOM
  • This leads to a dynamically changeable structure of an XHTML document
  • JS uses the DOM API to: add/remove elements, change content, respond to events
2 CSS — Cascading Style Sheets
CSS is a client-side language for defining the layout of markup language elements. Simplifies uniform design and its maintenance.
Inclusion

How to Include CSS

  • Embedded directly in (X)HTML
  • Linked externally: <link rel="stylesheet" href="main.css">
  • Key benefit of external: editing ONE file changes the layout of ALL associated pages
Structure

Selectors & Properties

  • CSS file contains selectors (HTML tags or custom names)
  • Each selector has properties with values
  • Selectors refer to elements by: tag name, class (.name), ID (#name)
/* tag selector */ p { font-size: 16pt; } /* class selector */ .top { padding-top: 20px; } /* ID selector */ #fat { font-weight: bold; }
CSS Properties Examples
