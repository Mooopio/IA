1 · What Is the Semantic Web?

Core Concept

The Semantic Web (also called Web 3.0, the Linked Data Web, and the Web of Data) is the next major evolution in connecting information on the internet.

Think of today's web as a library where books are written for humans — machines can't truly "read" them, only fetch them. The Semantic Web adds machine-readable meaning to data so computers can understand and act on it automatically.

🔑 Key Idea The Semantic Web is concerned with the meaning of data, not its structure — unlike relational databases or the regular web, which focus on structure.

Real-World Analogy

Imagine searching "design blogs" repeatedly. A traditional search engine just indexes words. But a Semantic Web system understands your pattern: it knows you care about design, so next time you search "computers," it surfaces results combining design + computers. The web learns your intent.

4 · Three Technical Standards (The Core Stack)

Must Know

RDF — Resource Description Framework

The data modeling language of the Semantic Web. All Semantic Web information is stored and represented in RDF. Think of RDF as the "grammar" — it defines how to describe things as subject–predicate–object triples (e.g., "The Eiffel Tower — is located in — Paris").

Must Know

SPARQL — SPARQL Protocol and RDF Query Language

The query language of the Semantic Web — the "SQL for RDF." It is specifically designed to query data across various heterogeneous systems. SPARQL lets you ask complex questions spanning multiple data sources.

Must Know

OWL — Web Ontology Language

The schema / knowledge representation (KR) language. OWL lets you define concepts composably — meaning each concept is carefully defined so it can be assembled in various combinations for many different applications. It is the "dictionary + grammar" of knowledge in the Semantic Web.

Composability: Carefully defined concepts that can be selected and assembled in various combinations with other concepts for many different purposes.

6 · Key Components: Metadata & Ontologies

Definition

Metadata

Metadata = data about data. For machines to understand existing human-readable data, we annotate it with metadata. Example: a photo file itself is data; the date taken, location, and photographer name are metadata.

Ontologies

Ontologies are documents that define relations among terms. They are the formal "dictionaries" of concepts and how they relate — enabling machines to reason about the domain.

Software Agents

Software agents process data on behalf of humans. Automated web services provide data. Together, they act as the "workforce" of the Semantic Web.

7 · User Interface Principles

  • UIs must make the richness of the underlying data model available at the interaction level.
  • Constraints at the interaction level should be minimized as much as possible.
  • Interfaces must be flexible enough to offer users as many ways as possible to interact with data.

8 · Mobile & Cloud Connection

The heavy usage of smartphones and cloud applications is driving the move toward accessing data from anywhere. The Semantic Web is expanding into smart TVs picking up user data, and smartphones accessing data on personal computers.

9 · Real-World Examples of Semantic Web

Applications

1. Google + Freebase (Metaweb)

Google acquired Metaweb, which runs Freebase — an open, semantically marked-up database. Similar in appearance to Wikipedia but focused on structured data.

2. GetGlue

A service where users "check in" to watching TV shows, reading books, listening to music — semantic context attached to media consumption.

3. Flipboard

Launched after the iPad (2010) — a magazine-reading application using the touch screen UI to deliver personalized, semantic content.

4. Facebook Open Graph

Announced in April 2010. Goal: enable publishers to "integrate their Web pages into the social graph." Each web page becomes an 'object' in Facebook's social graph — referenced and connected across profiles, posts, search results, and the News Feed.

5. Google Squared

The "holy grail" of search: ask a simple question in natural language, get a simple answer. Google Squared (launched 2009) adds additional structured information to search results — a semantic search engine for RDF documents that provides services to software agents.
